Integration

CloudScope plugs into the surrounding agent stack from runtime to policy to registry packaging.

kagent

tools:
  - name: cloudscope-mcp-server
    type: McpServer
    mcpServer:
      url: http://cloudscope-mcp-service:8000/mcp
      transport: http

agentgateway

rateLimit:
  requestsPerMinute: 60
accessPolicies:
  - tools: [search_docs, explain_concept, compare_cloud_tools]
    action: allow

agentregistry

name: cloudscope
category: cloud-native-learning
entrypoint: README.md
install:
  - kubectl apply -f kagent/agent.yaml

Getting Started

One product, three entry points: local Python, Claude Desktop, or a full kagent deployment.

cd cloudscope
python -m venv .venv
pip install -r mcp_server/requirements.txt
python mcp_server/server.py

Use stdio for local MCP clients or add --transport http to route through the gateway.

{
  "mcpServers": {
    "cloudscope": { "command": "python", "args": ["mcp_server/server.py"] }
  }
}

Claude Desktop is the fastest path for local evaluation because it avoids extra HTTP plumbing.

kubectl apply -f kagent/agent.yaml
kubectl apply -f agentgateway/gateway-config.yaml
kubectl get agent -n cloudscope

This path shows the complete story: deploy the MCP server, protect it with policy, and register the agent workflow.
